AI Open Source Picks is a curated shelf of open-source AI projects worth reading deeply, studying architecturally, and evaluating for real adoption.

It is not a newsfeed, and it is not a giant directory of repositories. It is a deliberately selective reading list for engineering judgment: we only keep the projects that genuinely shape agent execution environments, tool invocation, security boundaries, browser automation, remote development, and workflow construction.

Unlike the "Build Claw" section, this series does not focus on OpenClaw itself. Instead, it looks outward at the surrounding ecosystem: which pieces of infrastructure are worth borrowing from, integrating with, reusing, or even dropping directly into your own AI systems.

## What this section tries to answer

Every article here aims to answer four questions:

- What problem does this project actually solve?
- Is its architecture worth learning from?
- How does it relate to Agents, OpenClaw, and coding workflows?
- Is it worth the time to deploy, integrate, or extend?

## Current picks

- [OpenSandbox: Building Safe Boundaries for AI Agents](./opensandbox/)

## Suggested reading order

1. If you are building AI coding systems, code execution services, browser automation, or remote development environments, start with [OpenSandbox](./opensandbox/).
2. If you want to understand OpenClaw's own runtime, prompts, tools, and message loop, go back to [Build Claw](/en/build/).
3. If you care more about practical outcomes and scenario design, continue with [Lobster University](/en/university/intro).
